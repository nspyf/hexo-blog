react页面缓存：react-router-cache-route
316851756@qq.com
 2021-06-06 23:33:51  803 字  3 分钟  

今天处理了一下react页面缓存的问题。

之前没弄页面缓存，用户在页面间跳转的时候无法记录之前页面的状态比如滚轮高度等等，而且会重复向后端发请求，影响体验和性能。

之前也了解了一下react页面缓存的方案，如react-keeper，react-router-cache-route，还有用redux的。

由于引入redux比较复杂，于是重点考虑前两个方案。

react-keeper

首先尝试了一下react-keeper，但是npm install的时候就报错了

看样子是react版本不匹配？？？很奇怪

react-router-cache-route

于是转去尝试react-router-cache-route，react-router-cache-route是在react-router上扩展的，对于我的博客项目来说，改动很简单，把<Switch>改为<CacheSwitch>，把需要缓存的路由改为<CacheRoute>，就可以起到缓存页面的效果了。CacheRoute的属性when可以决定什么时候缓存，有三个可选值：”forward”,”back”,”always”,默认是”forward”。其它功能暂时没用到，以后需要再了解也不迟。

但使用时遇到点问题

  • 问题描述
    用户没有登录时个人页面有登录注册按钮，点击进入登录页面，登录成功后回到个人页面，此时个人页面会根据登录状态显示用户名、退出登录等按钮。也就是个人页面渲染的组件和登录状态相关。
    但是加了路由缓存之后，登录成功后回到个人页面，个人页面还是未登录状态的页面。

  • 问题解决
    查询相关文章后，发现react-router-cache-route有额外的生命周期componentDidCache 和 componentDidRecover，分别用在组件 被缓存 和 被恢复 时。但是没找到hooks相关的文章，而我的博客项目组件都不是class组件，全是函数组件配合hooks来使用。（为啥不用class组件，因为hooks简单易用，不需要考虑this等等🤤），然后自己去react-router-cache-route源码里浏览了一下，发现了和componentDidCache ，componentDidRecover对标的useDidCache和useDidRecover

    这正是我需要的，对于上面的问题，只需要使用useDidRecover调用相关useState改变登录状态即可。
    当然还有另一个解决方案：取消个人页面的路由缓存。这样做确实可以，但是考虑到文章开头提到的：“用户在页面间跳转的时候无法记录之前页面的状态比如滚轮高度等等，而且会重复向后端发请求，影响体验和性能“问题仍然有可能在后续迭代中在个人页面出现，其它的状态及数据仍然有缓存的需要，所以相比之下使用路由缓存及useDidRecover是更好的解决方案。
